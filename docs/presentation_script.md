# Сценарий видеопрезентации (3–4 минуты)

## 0:00–0:30 — задача

«PowerGuard AI помогает предварительно анализировать фотографии ЛЭП. Инспектор
загружает снимок, а система выделяет элементы конструкции и потенциальные
дефекты. Это ускоряет разбор больших объёмов фото, но итоговое решение остаётся
за специалистом».

## 0:30–1:00 — стек и архитектура

Показать схему из README. Назвать Streamlit, FastAPI, Ultralytics и две модели:
YOLO11n для скорости, YOLO11s для качества. Открыть `/docs`, выполнить `/health`
и `/api/v1/models`.

## 1:00–2:15 — демонстрация

1. Открыть вкладку «Инспекция».
2. Выбрать быструю модель, загрузить снимок, запустить анализ.
3. Показать boxes, число объектов, latency и скачивание результата.
4. Переключиться на точную модель и кратко объяснить компромисс.
5. Показать, что интерфейс отображает источник весов `custom`.

## 2:15–3:00 — качество

Открыть вкладку «Качество модели». Назвать mAP@0.5 = 0.902 и F1 = 0.87 при
confidence 0.439. Показать PR-кривую и матрицу ошибок. Отметить слабые классы:
`vibration_damper` и `polymer_insulators`.

## 3:00–3:30 — вывод

Сказать об ограничениях домена и обязательной ручной проверке. Следующие шаги:
больше данных слабых классов, SAHI для мелких объектов, ONNX/TensorRT для edge и
мониторинг качества после развёртывания.

Перед записью: включить custom-веса, подготовить 2–3 тестовых снимка, скрыть
личные данные и проверить доступность публичных ссылок.
